# [Django/DRF] 쿼리 1001개 → 1개로 최적화 (N+1 박살낸 후기)

주식 종목 검색 API를 개발하다가 문득 **"데이터가 많아지면 터지는 거 아님?"** 싶어서 더미 데이터 1,000개를 넣고 테스트를 돌려봤습니다.

결과는 처참했습니다.

**"목록 하나 불러오는데 쿼리가 1001방? 실화냐?"**

Django REST Framework(DRF)를 쓰면 숨 쉬듯이 겪는다는 **N+1 문제**였습니다.
`SerializerMethodField`를 믿고 막 짰다가 참교육 당하고, `annotate`를 통해 쿼리를 **1/1000**로 줄인 과정을 기록합니다.

---

## 1. 상황 (Before)

검색된 주식 리스트를 뿌려줄 때, **"내가 즐겨찾기 한 종목인가?(True/False)"**를 같이 내려줘야 했습니다.
아무 생각 없이 `SerializerMethodField`를 써서 구현했습니다.

### 범인 코드 (`serializers.py`)

```python
class StockSearchSerializer(serializers.ModelSerializer):
    # 👇 얘가 범인. for문 돌 때마다 함수 계속 호출함.
    is_favorite = serializers.SerializerMethodField()

    class Meta:
        model = Stock
        fields = ["id", "symbol", "name", "is_favorite"]

    def get_is_favorite(self, obj):
        # 😱 데이터가 1000개면 여기서 DB 접속도 1000번 함
        user = self.context['request'].user
        if user.is_authenticated:
            return FavoriteStock.objects.filter(user=user, stock=obj).exists()
        return False
```
이 상태로 테스트 스크립트(1,000개 조회)를 돌려본 결과입니다.

[여기에 아까 '실행 쿼리 수: 1001개' 나온 터미널 캡처 첨부]

조회 데이터: 1,000개

실행 쿼리: 1,001개 (목록 가져오는 1방 + 각 종목 체크하는 1,000방)

소요 시간: 0.26초 (데이터가 더 많았으면 서버가 뻗었을 겁니다.)

## 2. 해결 (After)
해결책은 간단합니다. View에서 DB를 긁어올 때 미리 계산(annotate)해서 가져오면 됩니다. 하지만 View만 고치면 안 되고 Serializer도 같이 고쳐야 합니다. (이걸 몰라서 삽질을 좀 했습니다.)

STEP 1. View 수정
Django ORM의 annotate와 Exists를 써서, DB 레벨에서 미리 True/False 값을 박아버립니다.

```python

# views.py
from django.db.models import Exists, OuterRef

# ...
if request.user.is_authenticated:
    # 서브쿼리: "이 유저가 이 종목을 즐겨찾기 했는가?"
    is_fav_subquery = FavoriteStock.objects.filter(
        user=request.user, 
        stock=OuterRef('pk')
    )
    # annotate로 'is_favorite_annotated'라는 가짜 필드를 미리 붙여서 가져옴
    queryset = queryset.annotate(is_favorite_annotated=Exists(is_fav_subquery))
```
### STEP 2. Serializer 수정 (중요)
View에서 아무리 준비해줘도, Serializer가 계속 get_is_favorite 함수를 돌리면 말짱 꽝입니다. **"야 계산하지 말고 아까 가져온 거(is_favorite_annotated) 갖다 써"**라고 명시해줘야 합니다.

```python

# serializers.py
class StockSearchSerializer(serializers.ModelSerializer):
    # 👇 MethodField 갖다 버리고 BooleanField로 변경
    # source='is_favorite_annotated' <- 이게 핵심
    is_favorite = serializers.BooleanField(source='is_favorite_annotated', default=False, read_only=True)

    class Meta:
        model = Stock
        fields = ["id", "symbol", "name", "is_favorite"]
    
    # get_is_favorite 함수는 삭제
```
## 3. 결과
코드를 고치고 다시 테스트를 돌려봤습니다.

[여기에 아까 '실행 쿼리 수: 1개' 나온 터미널 캡처 첨부]

조회 데이터: 1,000개

실행 쿼리: 1개 (1001개 → 1개 떡락)

소요 시간: 0.02초 (속도 10배 이상 빨라짐)

결론
N+1은 진짜 성능 킬러다. 데이터 적을 땐 모르니까 꼭 더미 데이터를 박고 테스트해보자.

View랑 Serializer는 한 세트다. View에서 annotate 했으면 Serializer에서도 받아먹게 세팅해줘야 한다. 안 그러면 쿼리는 줄어들지 않는다.

connection.queries를 찍어보는 습관을 들이자.
